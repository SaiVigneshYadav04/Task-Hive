Task-Hive - Routes Folder Description

The routes folder contains all the route files that handle the URL requests of the Task-Hive web application. These routes act as the connection between the frontend (templates) and the backend logic of the application.

Each route defines how the application responds when a user visits a specific URL or submits a form.

Main responsibilities of the routes folder:

1. Handling User Requests
   Routes receive requests from the user such as opening pages, submitting forms, or performing actions like posting or accepting tasks.

2. Connecting Frontend and Backend
   Routes take data from HTML forms in the templates and send it to the backend models for processing and database operations.

3. Rendering Templates
   Routes load and display the correct HTML pages using Flask’s render_template function.

4. Managing Application Logic
   Routes contain the logic required to process user actions such as user registration, login, posting tasks, viewing tasks, and accepting tasks.

In the Task-Hive application, the routes folder helps organize the backend functionality into separate files, making the code easier to manage, understand, and maintain.
